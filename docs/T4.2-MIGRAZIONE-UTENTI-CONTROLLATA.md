# T4.2 - Migrazione utenti JSON/CSV controllata

## Flusso implementato

Quando esiste `utenti.csv` ma non `utenti.json`:

1. parsing utenti da CSV;
2. creazione backup automatico in `backup/utenti_yyyyMMdd_HHmmss.csv.bak`;
3. scrittura `utenti.json`;
4. export CSV normalizzato via `SaveUsers()`;
5. audit log della migrazione.

## Fallback di sicurezza

Se la migrazione non parte o non è applicabile, il server carica comunque gli utenti dal CSV legacy senza alterare i file.

## Componenti

- `TimeClock.Server/Services/UserDataMigrationService.cs`
- integrazione in `TimeClock.Server/MainWindow.xaml.cs` (`LoadUsers`).
