# T2.3 - Pairing cross-day formalizzato

## Implementazione

Introdotto `PunchPairingService` (`TimeClock.Core/Services/PunchPairingService.cs`) come punto unico per accoppiare timbrature.

`WorkTimeCalculator.BuildPairsCrossDay(...)` ora delega a questo servizio.

## Regole edge-case

- `Entrata` seguita da un'altra `Entrata` senza `Uscita`: resta valida l'ultima entrata.
- `Uscita` senza `Entrata` aperta: ignorata.
- `Uscita` con timestamp <= entrata aperta: ignorata.
- `Entrata` finale senza uscita: ignorata.
- Pairing cross-day ammesso (es. 22:00 -> 06:00 giorno dopo).
