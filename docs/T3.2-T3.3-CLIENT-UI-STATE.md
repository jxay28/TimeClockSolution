# T3.2 + T3.3 - MVVM minimale e stato UI client

## Cosa è stato introdotto

- `ViewModelBase` con `INotifyPropertyChanged`.
- `RelayCommand` leggero (nessun framework esterno).
- `ClientStatusViewModel` con:
  - `StatusText`
  - `LastActionText`
  - comando `RefreshStatusCommand`

## Collegamento UI

In `TimeClock.Client/MainWindow.xaml`:

- `StatusTextBlock.Text` -> `{Binding StatusText}`
- `LastActionTextBlock.Text` -> `{Binding LastActionText}`

In `MainWindow.xaml.cs`:

- `DataContext = _statusVm`
- aggiornamento reale dello stato in:
  - selezione cartella
  - caricamento utenti
  - selezione utente
  - timbratura entrata/uscita
  - errori di scrittura

Risultato: feedback operativo più chiaro e meno ambiguità lato utente.
